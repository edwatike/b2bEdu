# Экраны системы (описание для заказчика)

Дата обновления: 05.02.2026

## Что это за документ
Здесь описано, какие экраны есть в системе, кто их видит и что на них можно сделать. Формат простой и без технических деталей.

## Роли
Модератор — управляет поиском поставщиков, черным списком, базой поставщиков и заявками пользователей.
Пользователь ЛК — создает заявки, получает результаты и ведет переписку.

## Вход и редиректы
- `/` — автоматический переход в вашу зону (модератор или личный кабинет).
- `/login` — вход через Яндекс.
- `/cabinet/login` — редирект на `/login`.
- `/cabinet/requests` — редирект на `/cabinet/requests/all`.
- `/moderator/users` — редирект на `/users`.
- `/moderator/suppliers` — редирект на `/suppliers`.

## Модераторская зона

**/moderator — Дашборд**
Кто видит: модератор.
Назначение: быстрый обзор системы и запуск парсинга.
Поля:
- ключевое слово
- глубина парсинга
- источник (Google, Яндекс, оба)
Кнопки:
- запустить парсинг
- быстрые переходы по карточкам (домены в очереди, поставщики, blacklist, задачи)
- обработать очередь
- проверить новых поставщиков
- управление blacklist
Блоки:
- карточки статистики
- последние запуски парсинга
- прогресс активного запуска

**/manual-parsing — Ручной запуск**
Кто видит: модератор.
Назначение: запуск парсинга по конкретному ключевому слову.
Поля:
- ключевое слово
- источник поиска
- глубина
Кнопки:
- запустить парсинг

**/parsing-runs — Запуски парсинга**
Кто видит: модератор.
Назначение: история запусков и управление ими.
Поля:
- поиск по ключевым словам
- фильтр по статусу
Кнопки:
- удалить выбранные
- удалить отдельный запуск
Блоки:
- таблица запусков
- чекбоксы массового выбора

**/parsing-runs/[runId] — Карточка запуска**
Кто видит: модератор.
Назначение: детальная работа с доменами конкретного запуска.
Поля:
- фильтры списка доменов
- выбор доменов для пакетной обработки
Кнопки:
- получить данные по доменам (ИНН/email)
- добавить домен в blacklist
- создать поставщика
- обновить поставщика
Блоки:
- список доменов по запуску
- результаты обогащения (ИНН/email)
- окно добавления в blacklist
- окно создания/редактирования поставщика

**/domains — Домены в очереди**
Кто видит: модератор.
Назначение: обработка доменов, которые еще не стали поставщиками.
Поля:
- поиск домена
- выбор доменов (чекбоксы)
Кнопки:
- получить данные (массово)
- в blacklist
- в поставщики
- обновить список
- пагинация (назад/вперёд)
Блоки:
- таблица доменов
- прогресс массовой обработки

**/keywords — Ключевые слова**
Кто видит: модератор.
Назначение: список ключевых слов для парсинга.
Поля:
- поиск по ключевым словам
- ввод нового ключевого слова
Кнопки:
- добавить ключевое слово
- обновить список
- удалить выбранные
- удалить одно ключевое слово
Блоки:
- список ключей с количеством URL и доменов
- раскрытие доменов по ключу

**/blacklist — Черный список**
Кто видит: модератор.
Назначение: управление запрещенными доменами.
Поля:
- домен
- причина (опционально)
Кнопки:
- добавить в blacklist
- удалить домен из blacklist
Блоки:
- таблица доменов с причинами и датой

**/suppliers — Поставщики**
Кто видит: модератор.
Назначение: база поставщиков и реселлеров.
Поля:
- фильтры и пагинация (в зависимости от выбора)
Кнопки:
- добавить поставщика
- открыть карточку поставщика
Блоки:
- таблица поставщиков

**/suppliers/new — Новый поставщик**
Кто видит: модератор.
Назначение: ручное добавление поставщика.
Поля:
- название
- ИНН
- email
- домен
- список email
- список доменов
- адрес
- тип (поставщик/реселлер)
- дополнительные поля Checko (ОГРН, КПП, адрес, финансы, соцсети)
Кнопки:
- сохранить
- отмена

**/suppliers/[id] — Карточка поставщика**
Кто видит: модератор.
Назначение: просмотр всех данных поставщика.
Кнопки:
- перейти к редактированию

**/suppliers/[id]/edit — Редактирование поставщика**
Кто видит: модератор.
Назначение: правка карточки поставщика.
Поля:
- название
- ИНН
- email
- домен
- список email
- список доменов
- адрес
- тип
- дополнительные поля Checko
Кнопки:
- сохранить
- отмена
- решить конфликт ИНН (привязать домен или обновить поставщика)

**/moderator/tasks — Задачи модератора**
Кто видит: модератор.
Назначение: задачи, созданные из пользовательских заявок.
Кнопки:
- обновить
- перейти в конкретный запуск парсинга
Блоки:
- список задач
- внутри задачи — связанные запуски парсинга

**/users — Пользователи**
Кто видит: модератор.
Назначение: управление доступом пользователей в личный кабинет.
Поля:
- поиск по username/email
- email нового пользователя
- организация
Кнопки:
- обновить список
- создать пользователя
- удалить выбранных
- переключатель доступа в ЛК
Блоки:
- список пользователей

**/settings — Настройки модератора**
Кто видит: модератор.
Назначение: ключи интеграций (OpenAI, Groq).
Поля:
- OpenAI API key
- Groq API key
Кнопки:
- сохранить

## Личный кабинет пользователя

**/cabinet — Создание заявки**
Кто видит: пользователь ЛК.
Назначение: создание заявки и добавление позиций.
Поля:
- название заявки
- список позиций
- файл для распознавания
Кнопки:
- распознать заявку
- сохранить черновик
- отправить в работу
- загрузить файл с позициями (вкладка «Файлы»)

**/cabinet/overview — Обзор**
Кто видит: пользователь ЛК.
Назначение: быстрый обзор по заявкам и коммуникациям.
Кнопки:
- создать заявку
Блоки:
- статистика «в обработке» и «готово»

**/cabinet/requests/all — Отправленные заявки**
Кто видит: пользователь ЛК.
Назначение: список заявок, отправленных модератору.
Кнопки:
- рабочий экран
- открыть в рабочем экране
- открыть поставщиков/переписку
- показать ещё

**/cabinet/requests/drafts — Черновики**
Кто видит: пользователь ЛК.
Назначение: черновики заявок.
Кнопки:
- создать заявку
- открыть черновик
- показать ещё

**/cabinet/requests/[id] — Детали заявки**
Кто видит: пользователь ЛК.
Назначение: поставщики и переписка по конкретной заявке.
Поля:
- список поставщиков с чекбоксами
Кнопки:
- отправить выбранным
- открыть карточку поставщика
- открыть переписку
- обновить список
Блоки:
- позиции заявки
- список поставщиков
- диалог переписки

**/cabinet/results — Результаты**
Кто видит: пользователь ЛК.
Назначение: итоговые поставщики по заявкам.
Поля:
- выбор заявки
Кнопки:
- открыть заявку
Блоки:
- список поставщиков со статусами

**/cabinet/messages — Почта**
Кто видит: пользователь ЛК.
Назначение: входящие и исходящие письма.
Поля:
- поиск
- фильтр по статусу
Кнопки:
- загрузить Яндекс.Почту
- открыть Webmail
- написать письмо
- ответить
Блоки:
- список писем
- просмотр письма
- демо‑режим без подключения

**/cabinet/settings — Настройки**
Кто видит: пользователь ЛК.
Назначение: настройки пользователя и доступа к интеграциям.
Поля:
- email
- app‑password
- Groq API key
Кнопки:
- сохранить
- сменить пароль
